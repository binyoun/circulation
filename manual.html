the system · process log

One page for the whole build, from simulator tuning to VFCD (21 to 27 September 2026). Read it top to bottom once; after that, jump to the phase you are in.

The one rule everything follows. Each cup is self-sufficient: it senses, computes its Five Elements colour, and glows with no network and no computer. Everything else (hub, TouchDesigner, room) is an optional layer above a system that already works. If a change would break that, do not make the change.

The cycle the light follows

상생, the generating cycle: each element feeds the next. When held cups sit adjacent on this circle, light travels between them. It is a fixed contract of the work.

What you do, phase by phase

  1. Now, before any parts arrive

    • Tune the interaction in the simulator until the mapping feels true. The knobs are named constants: element thresholds in fiveElements.ts, sync windows in sync.ts, PRESENCE and MARK_FADE_S in main.ts. Change a value, push, feel it live two minutes later.
    • Take TECHNICIAN_BRIEF.txt to the technician meeting.
    • Buy a real cupping cup set and order the parts list in the brief. Lead times are the schedule risk.
    • Hold a cup mouth-down on your palm for a minute; the posture decides the insert design.

  2. Bench week, when parts arrive

    • Install the Arduino IDE, add ESP32 board support, install the three libraries named at the top of cup.ino (SparkFun MAX3010x, Adafruit NeoPixel, CNMAT OSC).
    • One component at a time, breadboard first: LED ring alone, then pressure sensor alone, then MAX30102 alone, watching the Serial Plotter. Never debug two new parts at once.
    • The pin numbers in cup.ino assume a classic ESP32 dev board; a Xiao ESP32-C3 (the brief's recommendation) needs the config block remapped first. Do not flash blindly.
    • Run the two bench experiments from the brief: palm PPG under seal pressure (the design decider) and BMP280 inside a sealed cup. Uncomment RAW_LOG in cup.ino to print raw values. Keep all logs: they are the dataset's first entries.
    • Verify the afterglow on real hardware: grip, release, watch the ring hold and fade over about 40 seconds.

  3. MVP gate around week 4

    • One untethered cup: insert in a real cup, power bank, hand to colour, stable for 30 minutes.
    • Calibrate FSR_MIN / FSR_MAX (or the BMP280 equivalents) per cup from RAW_LOG values; set CUP_ID 1, 2, 3.
    • Only after this gate passes: build the other two.

  4. Three cups and the collective layer

    • Cups and laptop on one Wi-Fi network; set WIFI_SSID and HUB_HOST in each cup's config block.
    • Run npm run hub. Cups appear as /cup messages; every session is logged to logs/. Back those files up.
    • Rehearse the three-body choreography with real people and tune the sync windows against what real triads do.

  5. TouchDesigner and the room optional, first to cut

    • Work through touchdesigner/RECIPES.md in order. The hub already speaks to TD on port 9000; TD only listens and paints.
    • If TD or the room layer fails, the cups still work, by design.

  6. VFCD week 21 to 27 September

    • Bring: 3 cups + 1 spare cup body + 1 spare insert, charged power banks + spares, alcohol wipes (wipe cups between participants), the laptop with the hub, a printed copy of the contracts, a phone hotspot as backup Wi-Fi.
    • If Wi-Fi dies, the serial CSV fallback in cup.ino reads any cup over a USB cable.

Contracts that must not change

  • OSC, cup to hub, port 8000: /cup id pressure bpm confidence element r g b
  • OSC, hub to TouchDesigner, port 9000: /circulation/sync, /circulation/agreement, /circulation/room, /circulation/cup/<id>
  • The generating cycle: Wood feeds Fire, Fire feeds Earth, Earth feeds Metal, Metal feeds Water, Water feeds Wood.
  • No real suction marks on visitors' skin. The LED afterglow is the mark.
